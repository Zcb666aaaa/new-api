## 🚀 使用方法

### 1. 配置渠道
```yaml
渠道类型: CustomVideo
Base URL: https://api.example.com
API Key: sk-xxxxxxxxxxxxx
模型列表: custom-video-model
```

### 2. 配置价格
```yaml
模型: custom-video-model
基础价格: 0.1
其他倍率:
  seconds: 0.02
```

### 3. 调用 API
```bash
# 创建视频任务
curl -X POST https://your-api.com/v1/videos/create \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "custom-video-model",
    "prompt": "一只可爱的猫咪在草地上玩耍",
    "size": "1920x1080",
    "duration": 5
  }'

# 查询任务状态
curl -X GET https://your-api.com/v1/videos/task_abc123 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 🔍 技术细节

### 请求流程
```
客户端
  ↓ POST /v1/videos/create
new-api (验证、预扣费)
  ↓ 透传请求
{baseURL}/v1/videos/create
  ↓ 返回任务ID
new-api (存储任务)
  ↓ 返回标准响应
客户端
```

### 查询流程
```
后台轮询
  ↓ GET {baseURL}/v1/videos/{id}
上游服务
  ↓ 返回任务状态
new-api (更新任务)
  ↓
客户端查询
  ↓ GET /v1/videos/{task_id}
new-api
  ↓ 返回任务状态
客户端
```

### 状态映射
| 上游状态 | 标准状态 | 进度 |
|---------|---------|------|
| queued, pending | queued | 10% |
| processing, running, in_progress | in_progress | 50% |
| completed, succeeded, success | completed | 100% |
| failed, error, canceled, cancelled | failed | 100% |

## 🐛 故障排查

### 常见问题

1. **401 Unauthorized**
   - 检查 API Key 是否正确
   - 确认 Authorization 头格式正确

2. **404 Not Found**
   - 检查 Base URL 配置
   - 确认上游服务路径正确

3. **任务一直处于 queued 状态**
   - 检查任务轮询是否正常
   - 查看上游服务是否正常处理任务

4. **视频URL无法访问**
   - 确认上游服务返回的 video_url 可访问
   - 检查视频代理配置
