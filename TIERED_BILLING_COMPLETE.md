# 阶梯计费功能实现完成

## 功能简介

已成功实现基于输入token数量的阶梯计费功能。系统现在支持根据不同的token范围配置不同的输入/输出价格。

## 主要特性

1. **灵活的阶梯配置**
   - 支持无限数量的阶梯
   - 每个阶梯定义最小/最大token范围和对应价格
   - 支持输入和输出分别定价

2. **完整的UI界面**
   - 可视化的阶梯配置编辑器
   - 支持添加、编辑、删除阶梯
   - 直观的表格展示

3. **国际化支持**
   - 中文（zh）
   - 英文（en）

4. **与现有系统集成**
   - 兼容现有的分组倍率系统
   - 保持与其他计费方式（按量/按次/按秒）的隔离

## 使用示例

### 配置阶梯计费模型

1. 登录管理后台
2. 进入"设置" → "比率与模型倍率设置"
3. 选择"阶梯计费设置"标签页
4. 点击"添加阶梯计费模型"
5. 配置示例：

```
模型名称: claude-opus-4
阶梯1: 
  - 最小Token数: 0
  - 最大Token数: 32000
  - 输入价格: 4.0 ($/1M tokens)
  - 输出价格: 18.0 ($/1M tokens)

阶梯2:
  - 最小Token数: 32000
  - 最大Token数: 64000
  - 输入价格: 6.0 ($/1M tokens)
  - 输出价格: 22.0 ($/1M tokens)

阶梯3:
  - 最小Token数: 64000
  - 最大Token数: -1 (无限制)
  - 输入价格: 8.0 ($/1M tokens)
  - 输出价格: 26.0 ($/1M tokens)
```

6. 点击"确定"保存模型配置
7. 点击"保存更改"持久化到数据库

### 计费示例

假设用户在分组倍率为 1.0 的情况下使用上述配置的模型：

**场景1：小规模使用**
- 输入: 20,000 tokens
- 输出: 5,000 tokens
- 匹配阶梯: 第1阶梯 (0 <= 20000 < 32000)
- 计算:
  ```
  输入费用 = 20,000 * 4.0 / 1,000,000 = $0.08
  输出费用 = 5,000 * 18.0 / 1,000,000 = $0.09
  总费用 = $0.17 * QuotaPerUnit * 1.0
  ```

**场景2：中等规模使用**
- 输入: 50,000 tokens
- 输出: 10,000 tokens
- 匹配阶梯: 第2阶梯 (32000 <= 50000 < 64000)
- 计算:
  ```
  输入费用 = 50,000 * 6.0 / 1,000,000 = $0.30
  输出费用 = 10,000 * 22.0 / 1,000,000 = $0.22
  总费用 = $0.52 * QuotaPerUnit * 1.0
  ```

**场景3：大规模使用**
- 输入: 100,000 tokens
- 输出: 20,000 tokens
- 匹配阶梯: 第3阶梯 (64000 <= 100000)
- 计算:
  ```
  输入费用 = 100,000 * 8.0 / 1,000,000 = $0.80
  输出费用 = 20,000 * 26.0 / 1,000,000 = $0.52
  总费用 = $1.32 * QuotaPerUnit * 1.0
  ```

## 技术细节

### QuotaType 枚举值

| 值 | 类型 | 说明 |
|----|------|------|
| 0  | 按量付费 | 基于倍率（ModelRatio）计算 |
| 1  | 按次付费 | 固定价格（ModelPrice）|
| 2  | 按秒付费 | 按秒固定价格（ModelPricePerSecond）|
| 3  | **阶梯计费** | 基于token范围的阶梯定价（**新增**）|

### 数据存储

阶梯计费配置存储在系统配置表的 `ModelTieredPrice` 键中，格式为JSON：

```json
{
  "claude-opus-4": {
    "tiers": [
      {
        "min_tokens": 0,
        "max_tokens": 32000,
        "input": 4.0,
        "output": 18.0
      },
      {
        "min_tokens": 32000,
        "max_tokens": 64000,
        "input": 6.0,
        "output": 22.0
      },
      {
        "min_tokens": 64000,
        "max_tokens": -1,
        "input": 8.0,
        "output": 26.0
      }
    ]
  }
}
```

### 计费流程

1. **检测计费类型**
   - 调用 `service.IsTieredPriceModel(modelName)` 判断是否使用阶梯计费

2. **查找匹配阶梯**
   - 根据 `promptTokens`（输入token数）查找对应阶梯
   - 使用左闭右开区间：`[min_tokens, max_tokens)`

3. **计算费用**
   ```go
   inputCost = inputTokens * inputPrice / 1,000,000
   outputCost = outputTokens * outputPrice / 1,000,000
   quota = (inputCost + outputCost) * QuotaPerUnit * GroupRatio
   ```

4. **记录日志**
   - 在消费日志中标记"使用阶梯计费"
   - 记录实际使用的阶梯和价格信息

## 注意事项

1. **阶梯边界定义**
   - 使用左闭右开区间：`[min_tokens, max_tokens)`
   - 建议最后一个阶梯的 `max_tokens` 设为 `-1` 表示无上限

2. **与其他计费方式的关系**
   - 阶梯计费与按量/按次/按秒计费互斥
   - 同一模型只能使用一种计费方式
   - 优先级：阶梯计费 > 固定价格 > 倍率计费

3. **分组倍率**
   - 阶梯计费的最终费用仍然受分组倍率影响
   - 公式：`最终配额 = 基础费用 * QuotaPerUnit * GroupRatio`

4. **数据格式**
   - 价格单位统一为：美元/百万token
   - Token数量为整数
   - `-1` 在 max_tokens 中表示无上限
